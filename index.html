Hello, JavaScript

ECMAScript is JavaScript

In 1997, an international standard for JavaScript was formalized.

Browsers try to conform to this standard the same way they try to conform to standards provided by the W3C for HTML and CSS.

That standard is ECMAScript.

It's provided by ECMA International, a private, non-profit, membership-based standards organization originally founded in 1961.

“ECMA” stands for “European Computer Manufacturers Association”

All decisions are consensus-driven.

ECMAScript has versions, too.

Like CSS and HTML, ECMAScript has versions.

Version 4 was dropped for the same reasons XHTML 2 died on the vine.

Most of us code for version 3, but 5 is well supported, and 6 is on the way.

jQuery is written in JavaScript

If you're just getting into this space, you might be confused.

Think of them like ready-made themes for a WordPress site, or a set of InDesign templates: plug in and go.

Objects upon Objects

There are many kinds of objects.

Even null and undefined are objects!

Let's look at a few of them up close.

Complex objects

  • Object() is a generic JavaScript object: 
    boots = new Object();
  • Array() is great for lists: 
    toDo = new Array('Forge Sword', 'Slay Dragon', 'Ride off into Sunset');
  • Function() (more on this one soon!)

They're complex because they can contain lots of complicated data.

Primitive Values

Primitive Values are the simplest forms of information an object can represent.

  • String() is just text: 
    ballad = "I miss you, but I haven't met you yet.";
  • Boolean() has two values, true or false: 
    sword = true;
  • Number() for numberic values: 
    castles = 15;

null and undefined are primitives!

Complex Objects vs. Primitive Values

Did you notice something weird about how we made primitive values vs. complex objects?

boots = new Object(); //complex object

castles = 15; //primitive value

We use new Object() (or new Array(), etc.) when making a complex object.

Object() is a constructor function. More on those soon!

Object() as a Constructor

Object() makes simple values harder to compare and identify by wrapping them in an object.

A Number()/String()/etc. is an Object() first

castles = new Number(15);
typeof castles; // it's an Object()!

Objects() don't compare by their values

castles = new Number(15);
horses = new Number(15);

castles === horses; // compares two Objects(), not their numeric values

Primitives are Values.

Primitives are not objects. They're values.

castles = 15;
horses = 15;

castles === horses; // compares two numeric values

Danger: Type Coercion!

JavaScript makes assumptions about what types of thing you're comparing.

The == operator tries to guess what type you mean.

friends = 1; // a number
heart = true; // boolean value, also counts as "1"

friends == heart; // JS tries to guess what you mean

To avoid, we use === which never uses type coercion.

friends = 1; 
heart = true; 

friends === heart; // takes their types into account

Making Objects

Constructor Functions

Constructor functions are functions that make new objects.

var unicorn = new Object();
unicorn.color = "purple";
unicorn.wings = true;
unicorn.name = "Twilight Sparkle";
unicorn.getName = function() {return "Hi, I'm " + unicorn.name;};

Think of the Object() constructor as a cookie cutter for objects.

Object Literals

Object Literals are a more concise way to create objects on the fly.

var unicorn = {
  name: "Twilight Sparkle",
  color: "purple",
  wings: true,
  getName: function() {return "Hi, I'm " + unicorn.name;}
};

This is the preferred way to make objects 'cuz it's so tidy.

Prototypal Inheritance

Objects can have babies via constructor functions.

When they do, they pass on their methods and properties to the babeh objects.

// making a Unicorn() constructor function!
var Unicorn = function(name, color, wings) { 
  this.name = name;
  this.color = color;
  this.wings = wings;
  getName = function() {return "Hi, I'm " + unicorn.name;};
};

var twilightSparkle = new Unicorn('Twilight Sparkle', 'purple', 'true');

typeof twilightSparkle //object

twilightSparkle.constructor //unicorn

This is called prototypal inheritance

First-Class Objects

An object is first-class if it can be:

  • Constructed on the fly
  • Passed as a parameter
  • Returned from an operation
  • Assigned to a variable

Let's talk about another first-class object.

Functions

Functions are objects!

What's more, they are first-class objects!

Meaning we can make 'em, pass 'em, return 'em, and assign 'em as variables!

There are 3 ways to make a function

// function constructor
var addConstructor = new Function('x', 'y', 'return x + y'); //last parameter is the function logic

// function statement
function addStatement(x, y) {
  return x + y; 
}

// function expression
var addExpression = function(x, y) {
    return x + y;
};

Functions do stuff

Functions can be used to construct objects (as we've seen), return a value, or just run code.

Think of them as the verbs of JavaScript.

Returning from a function

return lets us return a value or object from a function.

function redHen() {
    return {
        laidEggs: "5 eggs",
        poops: "5 times a day"
    }
    console.log("Plot against the humans."); // doesn't happen, after return
}

var Henrietta = redHen(); // create an object via a function

It also stops the execution of the rest of the function.

Higher-order Functions

Functions that do stuff to functions are called “higher order” functions.

Let's see an example of a function that does stuff to other functions.

Callbacks

Callbacks are used by higher-order functions.

A callback is when you pass a function to another function to be executed when it's finished.

$("#goodbye").click(function() { //anonymous function passed as a parameter to click function
  alert("Holla back, y'all!");
});

Here, click() is a higher order function.

Functional Programming

A style of programming characterized by:

  • Abstraction of frequently used functions to simplify their use.
  • Nesting functions within functions, we'll talk about closures.
  • Passing functions as arguments to other functions, see callbacks.

jQuery is a great example of functional programming.

Very Important Objects

The Global Object: God of objects

When you create a function or a variable, it automatically belongs to an object.

By default, everything belongs to the Global Object.

(Note the capitilization for future reference.)

In the browser, the window is that Global Object.

It is also called the Head Object

It is inferred: window.alert("OMG!"); is the same as alert("OMG!");

The jQuery object, AKA $

By default, jQuery uses "$" as a shortcut for "jQuery"

That means this: $("#id").hide();

Is the same as: jQuery("#id").hide();

The jQuery object wraps a selector, allowing you to perform jQuery methods on it.

Scope and Variables

Global until proven otherwise

By default, all variables are global.

cat = "An adorable pet that purrs cutely.";

Imagine what a bad scene it would be if you were using some JavaScript on the page that also defined cat!

animalpedia = function() {
  cat = "A furry mammal with sharp teeth and claws.";
  console.log(cat);
}

Scoping Variables

If we put var infront of our variable, we scope it to its function.

This is called declaring a variable.

cat = "An adorable pet that purrs cutely."; //global cat

animalpedia = function() {
  var cat = "A furry mammal with sharp teeth and claws."; //local cat
  console.log(cat);
}

Closures

Powerfully Private

When variables are scoped to a function, only the function and sny functions it contains can access them.

Closures happen when you nest a function inside a function…

…and then use the variables of the containing function!

A Closure in Action

function princess() {
  var dragons = ["Great Blue", "Glitterfang", "Fafnir"]; // there are 3 dragons to slay

  return {
    slayDragon: function() {
      return dragons.pop(); //pop gets and removes the last array item
    }
  };
}

var littleGirl = princess(); //littleGirl is now secretly a princess

// Now she can slay dragons no one can see.
littleGirl.slayDragon(); 
littleGirl.slayDragon(); 
littleGirl.slayDragon();

this

When you make a function, a this is born.

this is like a variable that refers to the object containing the function.

var ourObject = {
  func1: function() {
    console.log(this); // logs ourObject
  }

ourObject.func1();

By default, this refers to the Global Object (because it's the big catch-all).

Hoisting

The function or the egg?

You can use a function statement in a script before you define it, like so:

var crow = function() {
    caw(); // caw() hasn't been defined yet, but you can still invoke it.
    function caw() {console.log('Caw caw caw!');}
}(); // invoke, remember the parentheses?

More reading on hoisting.

Recursion

Functions calling themselves

Recursion is when a function calls itself. See this example from Cody Lindley:

var countDownFrom = function countDownFrom(num) {
  console.log(num);
  num--;//reduce num by 1
  if (num < 0){return false;} // if num < 0 return function with no recursion
  countDownFrom(num);
};
countDownFrom(5); // kick off the function, which logs separately 5,4,3,2,1,0

APIs

What's an API?

An Application programming interface (API) is a set of rules different programs can use to talk to each other.

There are many APIs available to you.

Some, like the Web Audio API and Canvas API, are browser APIs.

Others, like Twitter's API, are provided so developers can interact with a company's data.

The DOM and jQuery

Every time you use JavaScript to manipulate the contents of a page, you're using the DOM, or Document Object Model API.

Every element in the HTML tree has a corresponding object available to JS called a node

jQuery provides normalized methods for getting at and manipulating the DOM.

Revisiting the Global Object

In the browser, our Global Object window always has a document node: the HTML document.

Remember this from earlier?

$('p'); // jQuery gets p

document.querySelector('p'); // Regular ol' JS gets p

So document.querySelector('p'); is the same as window.document.querySelector('p');
